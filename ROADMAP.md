# ROADMAP — Zukunft (NICHT MVP)

> Alles hier ist **bewusst außerhalb des MVP** (siehe [`docs/prd.md`](./docs/prd.md)).
> Nichts davon wird umgesetzt, bevor es als eigenes Issue in einem künftigen Milestone
> geplant ist. Zweck dieses Dokuments: die große Vision festhalten, **ohne** dass sie in
> den MVP leakt.

Grobe Reihenfolge (je eigene Pipeline-Runde, eigener Milestone):

1. Mehrere Listen / Kategorien
2. Kleines Backend + Geräte-Sync (Stack dann wählen: Node oder Java)
3. Bestände / Inventar von Items tracken
4. Rezepte + wöchentliche Essenspläne → generieren die Einkaufsliste
5. Angebote von Läden einbeziehen
6. Smart-Home-Integration

Noch nicht eingeordnete Ideen kommen als Issue mit Label `idea`/`backlog` — **nicht** als
„geplant" in dieses Dokument.
